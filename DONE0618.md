# 0618 完成项

## fix(core): scope supervisor loop tools by workflow phase

### 问题
`supervisor_analyse` 阶段 Supervisor 拿到 `read_file`/`grep`，导致自我探索循环，永远不执行 `transition("worker_do")`，Worker 不被调动。

### 修改文件

| 文件 | 变更 |
|------|------|
| `packages/core/src/resolve-effective-tools.ts` | 新增 `WorkflowPhase` 类型导入；用 `SUPERVISOR_LOOP_ANALYSE_TOOLS`/`CHECK`/`INTERVENE`/`DEFAULT` 四组 phase 集合替换单 `SUPERVISOR_LOOP_TOOLS`；新增 `supervisorLoopToolsForPhase()` 辅助函数；`resolveEffectiveTools()` 改用 phase-scoped 过滤；`ResolveEffectiveToolsOpts` 新增 `workflowPhase` 字段 |
| `packages/core/src/engine.ts` | `submit()` 签名新增 `workflowPhase?: WorkflowPhase`；用 `buildSupervisorLoopModePrompt()` 生成 phase 特化的 system prompt（analyse/check/intervene 各有独立指引）；新增 `phaseMaxTurns` 限制（analyse 2 turn, intervene 1 turn）；透传 `workflowPhase` 到 `resolveEffectiveTools()` |
| `packages/core/src/dual-agent-runtime/runtime.ts` | `submit()` 签名新增 `workflowPhase?: WorkflowPhase`；透传给 `engine.submit()` |
| `packages/core/src/workflow-coordinator/coordinator.ts` | `runSupervisorAnalyse()`/`runWorkerDo()`/`runWorkerReport()`/`runSupervisorCheck()`/`runSupervisorIntervene()` 各自传入当前 phase 给 `submit()` |

### Phase 工具集

| Phase | 工具 |
|-------|------|
| `supervisor_analyse` | `get_goal`, `list_dir` |
| `supervisor_check` | `get_goal`, `list_dir`, `read_file`, `grep` |
| `supervisor_intervene` | `get_goal` |
| 默认（无 phase） | `get_goal` |

## chore(core): harden workflow phase tool scoping (工程加固)

### 变更

| 文件 | 变更 |
|------|------|
| `packages/core/src/interface.ts` | 同步 `CoreEngine.submit` 接口签名为 `(userInput, agentConfig?, role?, mode?, workflowPhase?)`；新增 `WorkflowMode`、`WorkflowPhase` 类型导入 |
| `packages/core/src/engine.ts` | 将 `phaseMaxTurns` 内联逻辑抽为模块级 `resolvePhaseMaxTurns()` 辅助函数；`tools.filtered` 日志新增 `workflowPhase` 字段；`supervisor_analyse` prompt 增加硬停止条件 "Do not call tools repeatedly. If shallow orientation is insufficient, state assumptions in the plan and let the Worker inspect details." |
| `packages/core/__tests__/resolve-effective-tools.test.ts` | **新建**：5 个单元测试覆盖 analyse/check/intervene/default/worker 各场景的 phase-scoped 工具过滤 |
| `packages/core/__tests__/workflow-coordinator.test.ts` | 新增 1 个集成测试，验证 `runWorkflow()` 向 submit 传递正确的 phase 参数 |

### 新增测试

| 测试 | 覆盖内容 |
|------|----------|
| `resolveEffectiveTools — phase-scoped supervisor loop > supervisor_analyse: only shallow planning tools` | analyse 阶段仅 `get_goal` + `list_dir`，不含验证/执行工具 |
| `> supervisor_check: verification tools` | check 阶段含 `read_file`/`grep`，不含执行工具 |
| `> supervisor_intervene: minimal tools` | intervene 阶段仅 `get_goal` |
| `> no workflowPhase defaults to conservative` | 无 phase 时默认仅 `get_goal`，防止调用点遗漏 phase 参数退化 |
| `> Worker loop with workflowPhase: unaffected by supervisor phase rules` | Worker 的工程工具不受 Supervisor phase 限制影响 |
| `WorkflowCoordinator > passes the correct workflowPhase to each submit call` | 主路径 submit 调用携带正确的 phase |
